自我介紹 & 動機

Hello，我是 Emily，一名軟體工程師，對於學習新知識與技能充滿熱情。一開始對程式開發感興趣是源於想解決日常生活中的需求，所以參加勞動部提供為期半年的訓練課程，在課程期間學習 C# 和 Java 程式語言，在結訓前與團隊成員共同開發購物網站，我主要負責系統分析與設計、實作網頁的前端和後端。隨後轉職為軟體工程師，並負責實時資料流處理、資料庫存取以及即時數據分析等工作。 在結束上一段工作後，因為對 AI 領域感興趣，報名參加短期的 AI 課程，並對圖像辨識及神經網路有基本的理解，同時也有實作智慧圖像自助結帳工具，進一步實踐 AI 的技術。 除了專業領域的學習，我同時也在學習繪畫和日文，並計劃加強口說能力。為提高訓練口說的效率，我打算開發跟讀練習的網站，並透過 AI 進行語調和音高分析，幫助使用者辨識其語調和發音是否正確，以及提供改善建議，此外也想更深入了解模型架構的設計原理 ，因此這是我想參加深度學習 Bootcamp 的主要動機。

技能 & 作品

在工作期間，我主要使用 C# 進行開發，並遵循物件導向程式設計（OOP）原則進行系統開發。此外，我使用多執行緒技術來處理多工任務，並利用面向切面程式設計（AOP）技術來進行錯誤處理。另外為了確保測試環境與真實環境一致，並避免對主機系統造成污染，我使用 VMware 虛擬化技術搭建測試環境。
Checkout Bakery 智慧圖像自助結帳該系統主要使用 Python 開發，並以 Line Bot 作為自助結帳的介面。為了進行麵包圖像辨識，我們使用 LabelImg 標註軟體對三種類型的麵包進行標註，並使用 TensorFlow 的 Keras 來建立與訓練模型。最初，我們嘗試使用 YOLOv3 和 CNN 模型進行麵包圖像辨識，但準確率未達預期，低於 90%。隨後，我們改用了 ResNet 模型，並成功將辨識準確率提升至 92%。在此基礎上，我們進一步優化了資料集篩選，最終訓練出的 ResNet 模型達到 95% 的準確率。
DeWay電子商務平台主要使用 C# 開發，前端 UI 設計則使用 Bootstrap 進行頁面布局與響應式設計，並透過 jQuery 實現各項功能交互。整體架構上，我們採用了 MVC設計模式，將商業邏輯與表示層進行有效分離，確保各層級職責清晰。資料庫方面，我們選用 MSSQL 作為關聯式資料庫管理系統，並依照資料庫正規化原則進行設計，以提高資料存取效率並維持資料的一致性和完整性。

時間規劃

在正式進入學習之前，我計畫用兩週時間複習向量與矩陣的基本運算，為了更好掌握這些概念，我將使用 Numpy 進行練習，熟悉矩陣運算的觀念。 接下來第一階段將專注於數學理論與神經網路基礎。前四週將聚焦於線性代數的核心概念，包括線性獨立、逆矩陣與行列式、正交矩陣與矩陣分解。同時，我將學習神經網路的基本理論，例如激勵函數與損失函數的設計、前向傳播和反向傳播的運作原理，了解深度學習模型的核心架構。在第一階段的後四週，我將轉向實踐操作，練習使用Python 實現激勵函數與損失函數的程式碼，並搭建神經網路的前向與反向傳播演算法。此外透過 PyTorch，練習基礎深度學習模型，為後續進階應用做好準備。 第二階段的重點在於人工智慧應用的落地實踐與特徵工程技術。在這段時間裡，我將學習如何搭建 AI 應用服務系統，熟悉雲端服務與網站建置的技術，以了解如何將模型整合到實際應用中。此外，我還計畫學習特徵工程相關技術，提升對數據處理與分類器設計的理解。最後進行模型的優化和超參數調整，探索如何通過實驗與調整參數來提升模型準確度。 第三階段則將重點放在經典模型的應用與個人專案開發。在這段時間裡，我會練習深度學習中的經典模型，熟悉其應用場景與優化方法。同時，這一階段將以個人專案為主，我計畫設計並實現一個完整的 AI 應用專案，從模型設計到部署，充分體現所學知識的應用價值，並提升自己的實戰能力。 這份學習計畫涵蓋了基礎理論、實作練習與應用落地，目標是在掌握深度學習核心技術的同時，能夠將這些技術運用到實際場景中。通過這樣一個循序漸進的過程，我希望能夠扎實提升自己的理論基礎與實踐能力，為未來的人工智慧領域探索奠定堅實的基礎。

技術選擇的分析

如果這個領域能夠為尚未解決的需求提供更好的解決方案，那麼我認為這是值得投入的領域。接著，需要考慮的是該產業是否有足夠的人力需求。例如，某些產業已經被機械或自動化取代，但在AI領域，短期內仍需大量人力投入發展。最後，這個領域的生命週期應該處於發展階段，並且已經初步顯現出更明確的方向與成果。儘管AI的概念已經存在多年，但近幾年才取得實質的成果，顯示這是一個未來具潛力的領域。

負面情緒經驗

在製作 AI 課程的專題時，由於原先負責寫 Line Bot 的組員，在報告的三天前遲遲沒有回報進度，加上其他組員沒有寫程式的經驗，所以只好由我接下來這份工作，同時因為物件辨識準確率不如預期，因此僅剩三天的時間，讓我感到極大的壓力和焦慮。 處於焦慮的情況下，我會注意力無法集中，因此意識到情緒管理對於自己和工作都很重要，我必須先處理我的焦慮，所以我放下手邊的工作，專注在自己的呼吸，以此來釋放一部份的壓力，最後花時間評估讓自己感到最焦慮的部分，並優先處理最焦慮的工作讓情緒得以平復下來，同時注意力也漸漸恢復，在期限內完成專案。

網頁技能心得分享

在開發這份申請網頁的過程中，我選擇 React 和 Next.js 來構建靜態網站，並學習 React Hooks 的使用。在本次開發中，我將 useState、useEffect 和 useRef 用於事件監聽及頁面渲染後的動態操作（例如：Tabs 切換）。這些 Hooks 使我更有效地管理元件狀態和作用，從而提高開發效率。此外我利用 Next.js 的 SSG 和 SSR 的功能，讓我能實現頁面快速加載並提高 SEO 表現。在樣式開發方面，我使用以往未接觸過的 Tailwind CSS，來快速設計響應式布局，省去手動編寫 CSS 類名的時間，可以更快速的設計網頁頁面，最後，JSX 讓我能夠在 HTML 結構中直接編寫邏輯判斷，使得開發更加靈活，避免了大量繁瑣的模板程式碼，使得我能夠快速實現功能，並讓後續的維護和調整變得更加簡便。

不論是任何工作，其實都與社會群體息息相關，因為每一份工作都是社會運作的重要組成部分。就軟體行業而言，它與其他領域的不同之處在於，除了滿足人類需求外，還在深刻改變人們的生活方式。例如，社群軟體和影音平台的發展，不僅滿足了溝通和娛樂需求，還催生了新的職業類型，顯示出其對社會的深遠影響。然而，正如「水能載舟，亦能覆舟」所言，開發這些技術時，必須謹慎考量倫理問題，避免負面影響。
